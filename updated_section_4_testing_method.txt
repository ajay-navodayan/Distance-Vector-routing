4. Testing Method

4.1 Deployment (Docker Compose)
The topology is started using Docker Compose:

docker compose up -d --build

This command builds the router image and launches Router A, B, and C with the correct IP addresses, subnets, and neighbor settings defined in docker-compose.yml.

4.2 Convergence Verification
Convergence is verified by watching the router logs and the kernel routing tables:

docker logs -f router_a
docker logs -f router_b
docker logs -f router_c

The logs show each router's distance-vector table with distance and next hop entries.
Kernel tables can also be inspected:

docker exec router_a ip route show
docker exec router_b ip route show
docker exec router_c ip route show

4.3 Automated Testing
The automated test script is executed using:

.\test.ps1 -SleepSeconds 20

The script validates:
- Initial convergence (all routers learn all three subnets)
- Direct path preference (Router A keeps its direct route to 10.0.3.0/24)
- Failure handling (Router C stopped and routes through it removed)
- Recovery (Router C restarted and routes reconverge)
